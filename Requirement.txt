Client: Sales Analytics Platform

🎯 Business Problem

Currently:

Sales data is scattered across systems

No real-time visibility into pipeline and revenue

Hard to track sales team performance

No centralized reporting

We need:
👉 A scalable data pipeline + analytics platform
👉 Clean, transformed, and modeled data
👉 Interactive dashboards for decision-making

🏗️ Architecture Requirement

| Layer          | Tool                                |
| -------------- | ----------------------------------- |
| Ingestion      | Azure Data Factory                  |
| Raw Storage    | Azure Data Lake Storage (ADLS Gen2) |
| Transformation | Azure Databricks (PySpark)          |
| Data Warehouse | Azure Synapse Analytics             |
| Visualization  | Power BI                            |



Data Pipeline Requirements
1. Ingestion (ADF)

Build pipelines to:

Ingest all CSV files into ADLS (Raw Layer)

Schedule daily refresh

Handle schema validation

Log failures

👉 Output:

/raw/accounts/
/raw/sales_pipeline/
/raw/products/
/raw/sales_teams/
2. Data Lake Design

Create 3 layers:

/raw       → Original data
/processed → Cleaned data
/curated   → Business-ready data
3. Transformation (Databricks)
🔹 Step 1: Data Cleaning

Handle null values

Remove duplicates

Standardize column names

Convert date formats

🔹 Step 2: Data Modeling

Create Fact & Dimension Tables

✅ Dimension Tables

dim_accounts

dim_products

dim_sales_team

dim_date

✅ Fact Table

fact_sales_pipeline

🔹 Step 3: Business Logic

Implement:

Deal conversion rate

Revenue calculation

Sales cycle duration

Stage-wise funnel tracking

4. Data Warehouse (Synapse)

Load curated data into Synapse

Use star schema

Optimize with:

Partitioning

Indexing
